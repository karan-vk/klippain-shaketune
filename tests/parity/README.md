# Native/Python parity tests

Standalone (no pytest) parity checks that the compiled Rust `_core` native extension produces
bit-for-bit / tightly-tolerant identical results to Shake&Tune's pure-Python fallback
implementations, for the four numeric kernels the native module implements:

- **kernel 1** - `dir_speed_spectrogram` (vibrations profile direction/speed mapping)
- **kernel 2** - `spectrogram` (input shaper / belts comparison PSD spectrogram)
- **kernel 3** - `.stdata` read/write (v1 zstd+JSON-lines, v2 zstd binary)
- **kernel 4** - `klipper_psd` (Klipper's own `calc_freq_response` PSD math), plus an
  end-to-end "does the recommended input shaper agree" check. Requires a Klipper checkout.

These scripts use only `assert`/`numpy.testing` - no `pytest` dependency - so they run with a
plain `python3` as long as `numpy` and `zstandard` are installed.

## Running

1. Build the dev native extension (drops a `.so` at
   `shaketune/native/lib/dev/_core.abi3.so`, which the loader always prefers):

   ```bash
   scripts/build_native_dev.sh
   ```

   If this step is skipped, or fails, the runner still works: every kernel that needs the
   native module reports `SKIP` (not `FAIL`) instead of crashing.

2. Generate the deterministic corpus (fast by default; pass `--big` to also generate the
   1,000,000-sample file used for a heavier spectrogram check):

   ```bash
   python3 tests/parity/gen_corpus.py [--big]
   ```

3. Run the parity checks:

   ```bash
   python3 tests/parity/run_parity.py [--klipper-dir /path/to/klipper] [--big]
   ```

   `--klipper-dir` is only required for kernel 4 (it needs Klipper's own `shaper_calibrate`
   module as ground truth); every other kernel runs without it. Without native, kernel 4 also
   needs `shaketune.native.psd_compat` to have landed for full digest-gating coverage - if it
   hasn't yet, kernel 4 still runs the raw `klipper_psd` numeric comparison and simply notes
   that no digest allowlist is in effect.

The script prints a `NATIVE ACTIVE` / `FALLBACK (no native)` banner, then a per-kernel
PASS/FAIL/SKIP summary table, and exits non-zero if anything genuinely failed.

## Notes for the integrator

- This directory intentionally reimplements small readers (`read_v1_direct`,
  `reference_dir_speed_spectrogram`) from scratch rather than importing "the" reference from
  `shaketune` wherever a truly independent check matters (e.g. checking the native `.stdata`
  reader against a from-scratch JSON parse, not against `MeasurementsManager`, which itself may
  one day be wired to call into native code for reading).
- Kernel 2 works whether or not `shaketune.helpers.spectrogram.compute_spectrogram` has been
  wired to dispatch to native yet: it always compares one call made with native available
  against one call made with `SHAKETUNE_DISABLE_NATIVE=1` forced. Before wiring lands both calls
  take the fallback path (a trivial but harmless self-comparison); once wiring lands this
  becomes a real native-vs-fallback check with no changes needed here.
- Corpus files live in `_corpus/` (default), which is regenerated by `gen_corpus.py` and is not
  meant to be committed.
